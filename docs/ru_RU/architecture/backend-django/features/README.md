# Features (фичи)

## Обзор

Проект использует feature-based подход к организации кода. Каждая фича — это самостоятельный модуль со своими моделями, сериализаторами, представлениями и тестами.

## Структура фичи

```
features/
└── <feature_name>/
    ├── models.py
    ├── serializers.py
    ├── views.py
    ├── urls.py
    └── tests/
```

## Принципы

- Каждая фича изолирована и отвечает за одну бизнес-область
- Минимальные зависимости между фичами
- Собственные тесты внутри каждого модуля

Такой подход упрощает масштабирование и поддержку проекта.
